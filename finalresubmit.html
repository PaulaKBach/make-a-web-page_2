Basics of the Web

1. Web: A collection of html document. You move between html documents via hyperlinks.
2. html: "Hypertext Markup Language." Main type of content on the internet.
      a. html text content: What you see;
      b. html markup: content with the code language;
3. urls: How you refer to documents on the web.
4. http: protocol that the web uses.
5. web application

Components of the Working Web

Person -- Computer -- Internet --(http protocol) -- Servers

HTML Markup

An HTML Markup consists of tags which are designated by typing them between these symbols the less than and greater than symbols. Many tags have an opening tag and a closing tag. The contents of the tag appear between the opening and closing tags. The opening tag, content and closing tag are known as an "Element."

Tags can be inline or block. An inline tag changes the text on a line of text whereas a block tag usually ends by adding a blank line in the content.

Examples of Inline Tags
1) bold
2) italics
3) line break -- NOTE: This is also a "void" tag.
4) span
5) img
6) anchors

Examples of Block Tags
1) paragraph
2) div

It is not necessary to memorize the tags. Rather, it is important to learn how they operate. You can always look the tags up using a web search.

Fin.

Stage 1.2: Creating a Structured Document

Definitions
1. HTML: Hyper Text Markup Language. The standard markup language used to create web pages.
2. CSS: Cascading Style Sheets. Style sheet language used for describing the look and formatting of a document written in a markup language.
3. DOM: Document Object Model. A cross platform and language independent convention for representing and interacting with objects in HTML (and other markup languages). The modes of every document are organized in a tree structure, called the DOM tree.
4. JavaScript: Interactive language.

Structure of a Web Page

HTML langauge is generally used to form the structure of the webpage. For example, it defines the types headings and body the webpage will include. CSS controls the style of the page. That is, what does the page look like. What is the color of the background? Size of the font? The DOM is not a language at all really. It seems to be a system of organization. Is it like the card catalog? The index of a book? The DOM creates an outline of the webpage. By clicking on the "sideways triangle" beside each element, you can view increasingly greater detail about each element.

Boxes

Webpages are made up of a series of squares/rectangles. Although objects like a circle or a triangle may appear on the screen, the shape seen by the viewer is actually created by defining the style of the square or rectange. When designing a webpage, it makes sense to think of the page as a series of rectangles.


Fin.

Stage 1.3: Adding CSS Style and HTML Structure

HTML Document Tag Meanings:
1. "!DOCTYPE html": Tells browser the type of document to follow is an html document.
2. "html": Explains that everything to follow is html. This tag requires an end tag.
3. "head": This is where you add a link to the style file. The link to the style file should read: "less than sign" link rel="stylesheet" type="text/css" href="style.css" "greater than sign" Must be followed with an end tag.
4. Comments: To write comments in html ""
To write comments in CSS "/* */"

CSS style pages are designed so that the most specific rule is applied to every element. This is based on the concept of "inheritance." Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants. It relies on the document tree in the html document.

Text related properties are inherited and include:
      color
      font
      letter spacing
      line height
      list style
      text align
      text indent
      text transform
      visibility
      white space
      word spacing

Box related properties are not inherited. Box related properties include:
      background
      border display
      float & clear
      height
      width
      minimum/maximum margin
      minimum/maximum height/width
      outline
      overflow
      padding
      position
      vertical align

CSS Style basics: -- indent each property on its own line -- For styles applicable to all paragraphs: "p{ }" -- For classes ".description{ color: red; } -- Declaration: line of code -- Property: background-color -- Value: black

3 Ways to Include CSS Styling in a Document

1. Write CSS in "less than sign" head "greater than sign of html. Good for small projects only.

2. Link to a separate CSS file. Preferred.

3. Write style in-line with html. Not preferred. Results in repeat code. More difficult ot make stylistic changes.

Box Model

Boxes are made up of 4 parts: From the center out-- Content, Padding, Border, Margin (transparent). Some key CSS codes include: box-sizing: border box; For different web browsers use the following -webkit, -moz, -ms.

Box Sizing Key Take-Aways
1. HTML elements are boxes and each box has 4 components.
2. Because there are so many components to each box, it can often be hard to get the size of a box just right.
3. Thee are 2 techniques you can use to help deal with any issues.
      a. Set sizes in terms of percentages instead of pixels.
      b. Set the box-sizing attribute to border box for every element.

Box Positioning
1. Divs are block elements (as opposed to inline); so, they default to taking up the entire width of a page.
2. Adding the rule display: flex to the appropriate CSS will override this behavior and let divs appear next to each other.

Code, Test, Refine
1. Look for natural boxes.
2. Look for repeated styles and elements.
3. Write your html.
4. Apply styles. (From biggest to smallest)
5. Fix things.

Fin.
